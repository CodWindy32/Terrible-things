# Архитектура проекта

## Количество скриптов на игроке

**4–5 скриптов на объекте Player — это нормально** для Unity. Компонентный подход предполагает разбиение на небольшие, чётко отвечающие за одну задачу скрипты.

Текущее распределение:
- **Player (root):** FirstPersonController, PlayerStateMachine, CrouchHandler, LadderClimb
- **CameraHolder (ребёнок Player):** HeadBob, IdleBreathingEffect, FallImpactEffect

В более сложных играх на персонаже могут быть Animator, Health, Inventory, Combat и т.д., итого 10+ компонентов. Такой подход даёт:
- отдельные компоненты легко менять и тестировать;
- выполняется принцип единственной ответственности (SRP);
- меньше конфликтов при работе в команде.

## Принципы

- **SOLID:** каждый скрипт — одна зона ответственности
- **Состояния:** PlayerStateMachine — единый источник правды
- **Ввод:** Input System, все действия в одном InputActionAsset
- **UI:** ссылки на готовые объекты Canvas, без создания через код
